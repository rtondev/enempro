MELHORES PRÁTICAS NO USO DE REPOSITÓRIO DE CÓDIGO

1. ESTRUTURA
- Use nomes descritivos para repositórios, branches e commits
- Organize arquivos em pastas lógicas
- Arquivos essenciais: README.md, .gitignore, LICENSE, requirements.txt

2. COMMITS
- Commits pequenos e frequentes
- Mensagens claras: "tipo: descrição" (feat, fix, docs, refactor)
- Exemplos: "feat: adiciona login", "fix: corrige validação"

3. BRANCHES
- Nunca commitar diretamente na main
- Use branches: feature/nome, fix/nome
- Delete branches após merge

4. SEGURANÇA
- Nunca commitar senhas, tokens ou chaves de API
- Use .env para variáveis sensíveis
- Adicione .env ao .gitignore

5. .GITIGNORE
- Ignorar: venv/, __pycache__/, .env, *.log, arquivos de IDE

6. CODE REVIEW
- Revise código antes de merge
- Peça revisão de outros membros
- Teste mudanças localmente

7. DOCUMENTAÇÃO
- README com descrição, instalação e uso
- Mantenha documentação atualizada
